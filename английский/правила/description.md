# Правила английского языка

Раздел для интервального изучения правил английского языка.

## Как работаем

1. Пользователь сообщает, что прошел новую тему.
2. Для темы создается отдельный файл в этом разделе. Имя файла совпадает с названием темы, например `Present Perfect.md`.
3. В файл темы добавляются:
   - краткое описание правила;
   - что именно изучаем;
   - примеры;
   - задания;
   - история вопросов, ответов, ошибок и разбора.
4. В `lessons.md` добавляется или обновляется запись по теме:
   - ссылка на файл темы;
   - дата следующего повторения;
   - параметры SuperMemo SM-2: repetitions, interval, ease_factor;
   - последняя оценка ответа.
5. Один раз в день выбирается не больше одной темы, у которой `next_review` меньше или равен текущей дате.
6. После ответа пользователя тема оценивается, ошибки разбираются, а дата следующего повторения пересчитывается по SM-2.

## Шкала оценки SM-2

- 5 — идеально, без ошибок.
- 4 — правильно, но были небольшие сомнения или мелкие недочеты.
- 3 — в целом вспомнил, но были заметные ошибки.
- 2 — вспомнил частично, нужно повторить скоро.
- 1 — почти не вспомнил.
- 0 — не вспомнил совсем.

Для оценок ниже 3 серия повторений сбрасывается, а тема ставится на скорое повторение.

## Разделение ответственности

Детерминированные операции выполняются скриптами, чтобы расписание не зависело от интерпретации агента.

### Выбор темы для ежедневного повторения

Скрипт:

```bash
/home/alex/.hermes/scripts/english_review_select.py
```

Что делает:

- читает `lessons.md`;
- берет текущую дату в `Asia/Yekaterinburg`, если дата не передана явно;
- выбирает не больше одной темы с `next_review <= текущая дата`;
- если тем несколько, выбирает самую просроченную, а при равенстве — первую в таблице;
- возвращает JSON со статусом `due` или `none_due`.

Пример ручного запуска:

```bash
/home/alex/.hermes/scripts/english_review_select.py --date 2026-05-19
```

### Обновление даты после проверки ответа

Скрипт:

```bash
/home/alex/.hermes/scripts/english_review_update.py
```

Что делает:

- получает тему и оценку SM-2 от агента;
- пересчитывает `repetitions`, `interval_days`, `ease_factor`, `next_review`, `last_grade`, `updated`;
- обновляет строку темы в `lessons.md`;
- возвращает JSON со старым и новым состоянием.

Пример ручного запуска:

```bash
/home/alex/.hermes/scripts/english_review_update.py --topic "Subject and Object Questions" --grade 5
```

Можно проверить расчет без записи:

```bash
/home/alex/.hermes/scripts/english_review_update.py --topic "Subject and Object Questions" --grade 5 --dry-run
```

### Роль агента

Агент не должен вручную выбирать тему или считать дату следующего повторения, если доступны эти скрипты.

Агент отвечает за недетерминированную часть:

- подготовить объяснение и задания по выбранной теме;
- проверить ответ пользователя;
- разобрать ошибки;
- выставить качественную оценку SM-2 от 0 до 5;
- после оценки запустить `english_review_update.py`;
- записать историю вопроса, ответа и разбора в файл темы.

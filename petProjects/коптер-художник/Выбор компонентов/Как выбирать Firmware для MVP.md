# Как выбирать Firmware для MVP

Источник: [[Комплектующие MVP]], [[Выбор компонентов/Как выбирать Flight Controller для MVP|Как выбирать Flight Controller для MVP]], [[План разработки MVP]]

## Главный принцип

Firmware — это не комплектующая, а программная основа автопилота.

Её нужно выбирать вместе с flight controller, наземной станцией, GPS/compass, RC receiver, telemetry и будущими задачами автономности.

Для коптера-художника важны:

- стабильный полёт в режиме multicopter;
- понятная настройка первого MVP;
- GPS hold и waypoint missions;
- telemetry и логи;
- failsafe-сценарии;
- возможность будущей интеграции с companion computer;
- активная документация и сообщество.

## Кандидаты

### ArduPilot Copter

Статус: основной выбор для MVP.

Плюсы:

- зрелая экосистема для multicopter, rover, plane и payload-проектов;
- хорошо подходит для тяжёлых и нестандартных аппаратов;
- сильная поддержка mission planning, waypoint missions и MAVLink;
- подробные onboard logs для диагностики;
- гибкие failsafe-настройки;
- работает с Mission Planner и QGroundControl;
- подходит для будущего companion computer через MAVLink.

Минусы:

- много параметров, легко запутаться при первой настройке;
- требует аккуратной калибровки compass, accelerometer, ESC, radio и failsafe;
- для некоторых контроллеров нужно внимательно проверять конкретный target.

Почему подходит нам:

ArduPilot Copter лучше всего совпадает с задачей проекта: сначала стабильный летающий MVP, потом удержание позиции, waypoint missions, телеметрия, логи и будущая интеграция датчиков/управления распылением.

### PX4

Статус: сильная альтернатива, но не основной выбор на текущем этапе.

Плюсы:

- хорошо сочетается с Pixhawk-экосистемой;
- современная архитектура;
- удобен для research/autonomy;
- хорошая интеграция с QGroundControl;
- часто используется в developer platforms.

Минусы:

- для прикладного payload-MVP ArduPilot обычно проще найти готовые рецепты и обсуждения;
- часть нестандартных сценариев может потребовать больше разработки;
- переход между PX4 и ArduPilot потом потребует перенастройки параметров и проверок.

Когда рассматривать:

- если проект уйдёт в research-сторону;
- если будет сильная причина использовать PX4 tooling;
- если выбранная hardware-платформа лучше документирована именно под PX4.

### INAV

Статус: не основной выбор.

Плюсы:

- проще для некоторых FPV-сборок;
- удобен для fixed-wing/FPV-навигации;
- легче воспринимается, чем ArduPilot, для простых задач.

Минусы:

- хуже подходит для тяжёлого payload-коптера и будущей автономности;
- меньше фокус на промышленный/payload workflow;
- не лучший выбор для Pixhawk 6C Mini kit.

Когда уместен:

Если проект был бы FPV-крылом или простым навигационным коптером без серьёзной автономности. Для коптера-художника лучше не выбирать.

### Betaflight

Статус: не подходит для нашего MVP.

Плюсы:

- отличен для FPV racing/freestyle;
- быстрая реакция управления;
- простая экосистема для маленьких FPV-дронов.

Минусы:

- не ориентирован на waypoint missions и payload;
- не тот профиль стабилизации для тяжёлого hover-коптера;
- слабее для автономных сценариев, логов и mission workflow.

Для коптера-художника Betaflight не является подходящей основой.

## Рекомендация для MVP

Выбор на текущий момент:

`Pixhawk 6C Mini kit → ArduPilot Copter → Mission Planner / QGroundControl → MAVLink telemetry`

Причины:

- Pixhawk 6C Mini хорошо вписывается в ArduPilot/Pixhawk-экосистему;
- ArduPilot Copter лучше подходит для payload drones и нестандартных задач;
- Mission Planner удобен для первичной настройки, калибровок и анализа логов;
- MAVLink оставляет путь к companion computer и будущей автоматизации.

## Что проверить перед настройкой

- есть ли стабильная ArduPilot Copter firmware для выбранного Pixhawk 6C Mini;
- какая версия ArduPilot рекомендована для первого запуска;
- правильно ли выбран frame type: quadcopter и нужная геометрия;
- как подключены motor outputs;
- как подключены GPS/compass;
- как подключён ELRS receiver;
- как подключена telemetry;
- корректно ли настроен battery monitor через power module;
- работают ли safety switch и buzzer;
- включены ли нужные failsafe.

## Минимальный план настройки

1. Установить Mission Planner или QGroundControl.
2. Залить ArduPilot Copter на Pixhawk 6C Mini.
3. Выбрать тип рамы и раскладку моторов.
4. Откалибровать accelerometer и compass.
5. Настроить radio/receiver.
6. Настроить battery monitor.
7. Проверить motor order и направление вращения без пропеллеров.
8. Настроить failsafe: RC loss, battery failsafe, geofence при необходимости.
9. Проверить логи и вибрации на первых тестах.
10. Только после этого переходить к аккуратным полётам с пропеллерами.

## Чеклист выбора firmware

1. Поддерживает ли firmware выбранный flight controller?
2. Поддерживает ли именно multicopter/Copter, а не только Plane/VTOL?
3. Есть ли нормальная документация под Pixhawk 6C Mini?
4. Есть ли telemetry и onboard logs?
5. Поддерживаются ли GPS hold и waypoint missions?
6. Можно ли настроить failsafe под MVP?
7. Есть ли путь к companion computer/MAVLink?
8. Достаточно ли примеров и сообщества для отладки?

Для текущего проекта ответ почти везде лучше всего закрывает ArduPilot Copter.

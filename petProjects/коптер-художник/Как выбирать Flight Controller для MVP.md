# Как выбирать Flight Controller для MVP

Источник: [[Комплектующие MVP]], [[План разработки MVP]], [[Сметы MVP — варианты комплектующих AliExpress]]

## Главный принцип

Flight controller нужно выбирать не как отдельную плату, а как центр всей системы:

`flight controller → ArduPilot → питание → GPS/compass → RC receiver → telemetry → ESC/motors → future sensors`

Для первого MVP важнее не максимальная продвинутость контроллера, а понятная поддержка ArduPilot Copter, нормальная документация, простое подключение и низкий риск ошибок сборки.

## Что должен делать контроллер в нашем MVP

Минимально:

- летать на ArduPilot Copter;
- поддерживать quadcopter frame;
- иметь 4 motor outputs для ESC;
- подключать GPS + compass;
- подключать RC receiver;
- подключать telemetry radio;
- подключать power module / current sensor;
- сохранять логи;
- поддерживать failsafe;
- иметь понятную схему подключения.

Желательно с запасом:

- несколько UART для GPS, telemetry, receiver, будущих датчиков;
- I2C для компаса/дальномеров;
- CAN для будущего GNSS/RTK или периферии;
- барометр;
- хорошая виброустойчивость IMU;
- нормальный корпус или удобное крепление;
- активное сообщество и документация.

## Критерии выбора

### 1. Поддержка ArduPilot Copter

Это главный критерий.

Проверить:

- есть ли плата в списке поддерживаемых ArduPilot targets;
- есть ли именно прошивка Copter, а не только Plane/VTOL;
- есть ли инструкция по подключению motor outputs;
- есть ли понятная схема портов;
- нет ли известных проблем с этой платой в ArduPilot.

Если контроллер поддерживает только Betaflight/iNav или ориентирован на FPV racing, для нашего проекта он не подходит.

### 2. Простота интеграции

Для первого MVP удобнее контроллеры Pixhawk-экосистемы:

- стандартные разъёмы;
- понятные power modules;
- готовые кабели;
- понятное подключение GPS/compass;
- много инструкций для ArduPilot и Mission Planner.

Matek и похожие платы могут быть дешевле и мощнее за цену, но требуют больше ручной интеграции: пайка, распиновка, питание, настройка портов.

### 3. Процессор и запас ресурсов

Практические уровни:

| Класс | Пример | Оценка |
|---|---|---|
| Старый минимум | STM32F4 / Pixhawk 2.4.8 | годится для обучения, но не лучший выбор для долгого проекта |
| Нормальный современный уровень | STM32H7 / Pixhawk 6C Mini / Matek H743 | хороший выбор для MVP и будущих датчиков |
| Избыточный уровень | Pixhawk 6X и выше | хорошо, но дорого для первого MVP |

Для нашего проекта разумно целиться в STM32H7-контроллер, если бюджет позволяет.

### 4. Порты

Минимально нужны:

- один UART для GPS;
- один UART для telemetry radio;
- один UART или вход для RC receiver: CRSF/SBUS/PPM/PWM bridge;
- I2C для компаса, если он не идёт по GPS-разъёму;
- 4 PWM/DShot outputs для ESC;
- вход power module.

С запасом лучше иметь:

- дополнительный UART под companion computer;
- CAN под будущий GPS/RTK или smart peripherals;
- отдельный I2C под rangefinder;
- достаточное количество servo/PWM outputs под будущий клапан или mock spray.

### 5. Питание

Контроллер нельзя питать напрямую от 6S батареи.

Нужны:

- power module, который принимает напряжение батареи;
- стабилизированное питание контроллера;
- измерение напряжения батареи;
- измерение тока;
- понятная совместимость разъёмов.

Для 6S MVP обязательно проверить, что power module поддерживает:

- напряжение 6S: полный заряд 25.2 V;
- ожидаемый суммарный ток;
- разъёмы батареи: XT60 / XT90 / другое;
- токовый запас под реальные моторы.

### 6. IMU, вибрации и крепление

Для большого hover-коптера вибрации важны.

Проверить:

- есть ли виброразвязка контроллера или мягкое крепление;
- как плата крепится на раму;
- удобно ли ориентировать стрелку/ось контроллера;
- есть ли корпус;
- не будет ли плата стоять рядом с силовыми проводами и ESC.

Pixhawk-подобные контроллеры обычно удобнее физически монтировать. Голые платы типа Matek требуют аккуратного размещения.

### 7. Совместимость с RC

Если использовать RadioMaster Pocket ELRS, лучше сразу понять схему подключения:

- CRSF receiver по UART — предпочтительно;
- PWM receiver через PWM bridge — возможен, но менее элегантно;
- SBUS — тоже возможен, если приёмник его поддерживает.

Для ArduPilot лучше иметь понятную инструкцию именно под выбранный receiver.

### 8. Логи и отладка

Для MVP это важно: первые полёты нужно анализировать по логам.

Желательно:

- microSD для onboard logs;
- нормальная поддержка Mission Planner / QGroundControl;
- доступ к vibration logs;
- понятная загрузка прошивки;
- USB без нестабильных переходников.

## Кандидаты для нашего проекта

### 1. Pixhawk 6C Mini kit

Статус: основной кандидат для доступного, но нормального MVP.

Плюсы:

- современный Pixhawk-класс;
- обычно проще интегрировать с ArduPilot, чем Matek WING;
- часто продаётся комплектом с GPS и power module;
- понятнее для новичка в ArduPilot;
- хорошая база для waypoint missions и telemetry.

Минусы:

- дороже старого Pixhawk 2.4.8;
- нужно проверить, что в конкретный kit входят GPS, power module, safety switch, buzzer и кабели;
- Mini-формат может иметь меньше портов, чем полноразмерные контроллеры.

Что проверить перед покупкой:

- точная комплектация kit;
- поддержка ArduPilot Copter;
- количество UART;
- совместимость power module с 6S и током MVP;
- типы разъёмов;
- как подключить ELRS receiver.

Предварительная оценка: лучший баланс для первого серьёзного MVP.

### 2. Matek H743-WING V3

Статус: возможный бюджетно-гибкий вариант, но требует проверки для multirotor.

Плюсы:

- STM32H7;
- много портов;
- дешевле многих Pixhawk-комплектов;
- Matek обычно хорошо документирует платы;
- полезен для будущих экспериментов с периферией.

Минусы:

- WING-версия ориентирована на Plane/VTOL;
- может быть менее plug-and-play для обычного квадрокоптера;
- больше ручной интеграции;
- нужно внимательно проверить motor outputs, питание, разъёмы и ArduPilot Copter target;
- выше риск ошибки при первой сборке.

Что проверить перед покупкой:

- есть ли ArduPilot Copter target именно для этой платы;
- как подключаются 4 ESC;
- как подключается GPS/compass;
- как подключается power/current sensor;
- нужна ли пайка;
- как крепить плату на X500.

Предварительная оценка: хороший вариант для человека, готового разбираться в проводке и распиновке. Для первого спокойного MVP Pixhawk 6C Mini выглядит безопаснее.

### 3. Pixhawk 2.4.8

Статус: нижняя бюджетная граница / учебный вариант.

Плюсы:

- дешёвый;
- часто продаётся комплектом с GPS, telemetry, safety switch, buzzer;
- много старых инструкций;
- подходит для обучения Mission Planner и ArduPilot.

Минусы:

- старая аппаратная платформа;
- качество клонов сильно плавает;
- меньше запаса на будущие датчики;
- не лучший выбор для проекта, который планируется развивать.

Предварительная оценка: можно брать только если задача — максимально дешёвое обучение. Для основного коптера-художника лучше не выбирать.

### 4. Pixhawk 6X / Cube / более дорогие контроллеры

Статус: качественно, но может быть избыточно для первого MVP.

Плюсы:

- больше надёжности и запаса;
- лучше документация;
- больше портов;
- лучше подходит для будущей промышленной версии.

Минусы:

- заметно дороже;
- цена ошибки выше;
- для первого летающего MVP возможности могут быть избыточны.

Предварительная оценка: рассматривать позже, если проект подтвердит жизнеспособность.

## Предварительная рекомендация

Для текущего MVP с бюджетными моторами Cosmicrc 4108 логично не уходить в слишком дорогой контроллер, но и не брать совсем старый Pixhawk 2.4.8 как основную платформу.

Рекомендуемый путь:

1. Основной кандидат: `Pixhawk 6C Mini kit`.
2. Альтернатива, если хочется дешевле и гибче: `Matek H743-WING V3`, но только после проверки ArduPilot Copter и схемы подключения.
3. Учебный минимум: `Pixhawk 2.4.8`, если цель — просто дешёвый первый запуск ArduPilot.

Моя предварительная рекомендация: искать конкретный `Pixhawk 6C Mini kit` с GPS, power module и кабелями, затем сравнить его цену с Matek H743-WING V3 плюс отдельные GPS/power module/кабели. Часто kit выходит дороже на витрине, но дешевле по времени и риску интеграции.

## Чеклист перед покупкой

1. Есть ли официальная поддержка ArduPilot Copter?
2. Есть ли понятная схема подключения именно для multirotor?
3. Достаточно ли UART под GPS, telemetry, RC и будущие датчики?
4. Есть ли I2C/CAN для будущего расширения?
5. Поддерживает ли power module 6S и нужный ток?
6. Входит ли GPS/compass в комплект?
7. Входит ли safety switch / buzzer?
8. Есть ли microSD для логов?
9. Как подключается ELRS receiver?
10. Нужно ли паять или всё подключается кабелями?
11. Подходит ли размер и крепление для X500?
12. Есть ли реальные отзывы по ArduPilot, а не только PX4/Betaflight?

## Что выбрать следующим

После flight controller нужно выбрать связанные компоненты:

- GPS + compass;
- power module / current sensor;
- telemetry radio;
- RC transmitter и receiver;
- ESC под Cosmicrc 4108;
- пропеллеры под выбранную версию мотора.

Flight controller лучше выбирать вместе с GPS и power module, потому что kit может закрыть сразу несколько позиций и уменьшить риск несовместимости.

# Анализ района по OpenStreetMap

Небольшой консольный инструмент для первичной оценки адреса в Перми: школы, медицина, остановки общественного транспорта и вручную заданные важные места.

Данные берутся из открытых источников OpenStreetMap:

- Nominatim: поиск координат по адресу.
- Overpass API: поиск объектов вокруг найденной точки.

Важно: расстояния считаются по прямой, а не по пешеходному маршруту. Это подходит для грубого фильтра, но перед покупкой нужно открыть карту и проверить реальный путь до школы, переходы, тротуары и освещение.

## Запуск

```bash
python3 analyze_location.py "проспект Декабристов, 19"
```

По умолчанию результат выводится в JSON, чтобы его мог читать другой инструмент.

С радиусом 2 км:

```bash
python3 analyze_location.py "улица Космонавта Леонова, 93" --radius 2000
```

Если адрес геокодится неточно, лучше взять координаты точки с карты и передать их явно:

```bash
python3 analyze_location.py "улица Космонавта Леонова, 93" --lat 57.954990 --lon 56.182938 --radius 2000
```

С сохранением полного отчета в JSON:

```bash
python3 analyze_location.py "улица Космонавта Леонова, 93" --radius 2000 --json-out leonova-93-report.json
```

Старый человекочитаемый вывод для одного адреса:

```bash
python3 analyze_location.py "улица Космонавта Леонова, 93" --radius 2000 --format text
```

С пользовательскими точками:

```bash
python3 analyze_location.py "улица Космонавта Леонова, 93" --custom custom_places.example.json
```

Проверка нескольких адресов:

```bash
python3 analyze_location.py --batch addresses.example.txt --radius 2000
```

Batch-режим тоже выводит JSON. Для табличного просмотра:

```bash
python3 analyze_location.py --batch addresses.example.txt --radius 2000 --format table
```

Можно использовать CSV, если точные координаты уже взяты с карты:

```csv
address,lat,lon
Участок на Космонавта Леонова 93,57.954990,56.182938
```

```bash
python3 analyze_location.py --batch my-addresses.csv --radius 2000
```

## Формат пользовательских мест

Файл должен быть JSON-массивом:

```json
[
  {
    "name": "Работа",
    "category": "work",
    "lat": 58.010455,
    "lon": 56.229443
  }
]
```

Категорию можно назвать как угодно: `work`, `relatives`, `kids_activity`, `favorite_place`.

## Что показывает

- Ближайшие школы.
- Ближайшие детские сады.
- Колледжи и вузы отдельной категорией.
- Ближайшие больницы, клиники, врачей, стоматологии и аптеки.
- Ближайшие остановки автобуса, трамвая и другие транспортные точки.
- Пользовательские места из JSON-файла.

## Формат результата

Одиночный запуск возвращает JSON-объект:

```json
{
  "address": "улица Космонавта Леонова, 93",
  "resolved_address": "...",
  "location": {
    "lat": 57.95499,
    "lon": 56.182938,
    "map_url": "https://www.openstreetmap.org/...",
    "house_number": null,
    "osm_type": "way",
    "osm_id": 123
  },
  "radius_m": 2000,
  "distance_mode": "straight_line",
  "warnings": [],
  "summary": {
    "schools": {
      "title": "Школы",
      "nearest_distance_m": 1010,
      "count": 4,
      "counts_within_m": {
        "500": 0,
        "1000": 0,
        "2000": 4
      }
    }
  },
  "places_limit_per_category": 7,
  "places": []
}
```

`summary` считает все найденные объекты в радиусе. `places` содержит ближайшие объекты по каждой категории, количество регулируется параметром `--limit`.

Batch-режим возвращает объект с массивом `results`. Если отдельный адрес не удалось проверить, в его элементе будет поле `error`.

## Ограничения

- OpenStreetMap может быть неполным или устаревшим.
- Геокодер может найти улицу вместо конкретного дома или участка. В этом случае скрипт покажет предупреждение, а для проверки лучше использовать `--lat` и `--lon`.
- У школ и больниц иногда нет имени в OSM, тогда показывается технический OSM ID.
- Для домов на окраинах расстояние по прямой может сильно отличаться от реального пешего маршрута.
- Инструмент не проверяет прикрепление к школе по прописке.
